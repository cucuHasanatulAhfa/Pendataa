implementasi Knime Gion Ratio

implementasi Knime Gion Ratio#

  1. CSV Reader (Input Data) Ini adalah gerbang awal. Node ini bertugas untuk membaca file dataset (kemungkinan data “Play Tennis” yang kamu hitung tadi) yang berformat .csv agar bisa diproses oleh KNIME.

  2. Table Partitioner (Pembagian Data) Node ini sangat penting dalam machine learning. Fungsinya adalah membagi dataset menjadi dua bagian:

Data Training: Biasanya 70-80%, digunakan untuk “mengajari” model.

Data Testing: Sisanya (20-30%), digunakan untuk menguji seberapa akurat model tersebut menebak data yang belum pernah dilihat sebelumnya.

  1. Color Manager & Color Appender (Visualisasi) Color Manager: Memberikan warna pada kategori tertentu (misalnya “Yes” jadi hijau, “No” jadi merah) agar hasil prediksi nantinya lebih mudah dibaca secara visual.

Color Appender: Menerapkan skema warna tersebut ke dataset yang akan masuk ke proses prediksi.

  1. Decision Tree Learner (Proses Belajar) Di sinilah “otak” dari workflow ini berada. Node ini menggunakan data training untuk membangun Pohon Keputusan.

Hubungan dengan hitunganmu: Di dalam node inilah algoritma (seperti C4.5 atau ID3) menghitung Information Gain dan Gain Ratio secara otomatis untuk menentukan cabang-cabang pohonnya.

  1. Decision Tree Predictor (Proses Menebak) Setelah model “pintar” karena sudah belajar di node Learner, model tersebut dikirim ke node Predictor. Di sini, model akan mencoba menebak label (Yes/No) pada data testing yang tadi dipisahkan oleh Table Partitioner.

  2. Scorer (Evaluasi Akurasi) Ini adalah tahap akhir untuk melihat performa model. Node ini akan membandingkan:

“Jawaban asli vs Jawaban tebakan model” Hasilnya biasanya berupa Confusion Matrix dan nilai Accuracy. Jika akurasinya 0.8 atau 80%, berarti modelmu sudah cukup bagus dalam memprediksi.
